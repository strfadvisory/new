# React Query API Architecture Guide\n\nThis guide explains how to use the new React Query-based API architecture in your MERN application.\n\n## 🏗️ Architecture Overview\n\nThe new API layer is organized into these layers:\n\n```\nsrc/\n├── api/\n│   ├── config.js          # API endpoints and query keys\n│   ├── client.js          # Axios instance with interceptors\n│   └── services/          # API functions\n│       ├── authApi.js\n│       ├── usersApi.js\n│       ├── rolesApi.js\n│       └── ...\n├── hooks/\n│   ├── QueryProvider.js   # React Query setup\n│   └── queries/           # React Query hooks\n│       ├── useAuth.js\n│       ├── useUsers.js\n│       ├── useRoles.js\n│       └── ...\n└── components/\n    └── AssociationsExample.tsx  # Example usage\n```\n\n## 🚀 Quick Start\n\n### 1. Setup (Already Done)\n\nThe QueryProvider is already wrapped around your App component in `App.tsx`:\n\n```jsx\nimport { QueryProvider } from './hooks/QueryProvider';\n\nfunction App() {\n  return (\n    <QueryProvider>\n      {/* Your app content */}\n    </QueryProvider>\n  );\n}\n```\n\n### 2. Using Query Hooks (GET requests)\n\n```jsx\nimport { useAssociations, useUsers, useRoles } from '../hooks/queries';\n\nfunction MyComponent() {\n  // Fetch associations\n  const { \n    data: associations = [], \n    isLoading, \n    error, \n    refetch \n  } = useAssociations();\n\n  // Fetch users with conditional loading\n  const { data: users } = useUsers();\n\n  // Fetch single role by ID\n  const { data: role } = useRole(roleId, !!roleId); // enabled when roleId exists\n\n  if (isLoading) return <div>Loading...</div>;\n  if (error) return <div>Error: {error.message}</div>;\n\n  return (\n    <div>\n      {associations.map(association => (\n        <div key={association._id}>{association.name}</div>\n      ))}\n    </div>\n  );\n}\n```\n\n### 3. Using Mutation Hooks (POST/PUT/DELETE)\n\n```jsx\nimport { useCreateAssociation, useUpdateAssociation, useDeleteAssociation } from '../hooks/queries';\n\nfunction MyComponent() {\n  const createMutation = useCreateAssociation();\n  const updateMutation = useUpdateAssociation();\n  const deleteMutation = useDeleteAssociation();\n\n  const handleCreate = async (formData) => {\n    try {\n      await createMutation.mutateAsync(formData);\n      // Success - queries will auto-refresh\n    } catch (error) {\n      // Handle error\n      console.error('Create failed:', error);\n    }\n  };\n\n  const handleUpdate = async (id, data) => {\n    try {\n      await updateMutation.mutateAsync({ associationId: id, associationData: data });\n    } catch (error) {\n      console.error('Update failed:', error);\n    }\n  };\n\n  const handleDelete = async (id) => {\n    try {\n      await deleteMutation.mutateAsync(id);\n    } catch (error) {\n      console.error('Delete failed:', error);\n    }\n  };\n\n  return (\n    <div>\n      <button \n        onClick={() => handleCreate({ name: 'New Association' })}\n        disabled={createMutation.isLoading}\n      >\n        {createMutation.isLoading ? 'Creating...' : 'Create'}\n      </button>\n    </div>\n  );\n}\n```\n\n## 📚 Available Hooks\n\n### Auth Hooks\n```jsx\nimport {\n  useProfile,           // Get user profile\n  useAuthUsers, fetch when component is visible\nconst [isVisible, setIsVisible] = useState(false);\nconst { data } = useAssociations({ enabled: isVisible });\n```\n\n### Manual Refetching\n```jsx\nconst { data, refetch, isFetching } = useAssociations();\n\nconst handleRefresh = () => {\n  refetch();\n};\n\nreturn (\n  <button onClick={handleRefresh} disabled={isFetching}>\n    {isFetching ? 'Refreshing...' : 'Refresh'}\n  </button>\n);\n```\n\n### Optimistic Updates\n```jsx\nconst queryClient = useQueryClient();\nconst updateMutation = useUpdateAssociation();\n\nconst handleOptimisticUpdate = async (id, newData) => {\n  // Cancel outgoing refetches\n  await queryClient.cancelQueries({ queryKey: QUERY_KEYS.ASSOCIATIONS.ALL });\n\n  // Snapshot previous value\n  const previousData = queryClient.getQueryData(QUERY_KEYS.ASSOCIATIONS.ALL);\n\n  // Optimistically update\n  queryClient.setQueryData(QUERY_KEYS.ASSOCIATIONS.ALL, (old) => \n    old.map(item => item._id === id ? { ...item, ...newData } : item)\n  );\n\n  try {\n    await updateMutation.mutateAsync({ associationId: id, associationData: newData });\n  } catch (error) {\n    // Rollback on error\n    queryClient.setQueryData(QUERY_KEYS.ASSOCIATIONS.ALL, previousData);\n  }\n};\n```\n\n### File Uploads\n```jsx\nconst uploadMutation = useUploadFile();\nconst createStudyMutation = useCreateReserveStudy();\n\nconst handleFileUpload = async (file, studyName, association) => {\n  const formData = new FormData();\n  formData.append('excelFile', file);\n  formData.append('studyName', studyName);\n  formData.append('associationName', association);\n\n  try {\n    await createStudyMutation.mutateAsync(formData);\n    // Success - reserve studies will auto-refresh\n  } catch (error) {\n    console.error('Upload failed:', error);\n  }\n};\n```\n\n## 🎯 Migration Guide\n\n### Before (Old Way)\n```jsx\n// ❌ Don't do this anymore\nconst [users, setUsers] = useState([]);\nconst [loading, setLoading] = useState(true);\nconst [error, setError] = useState(null);\n\nuseEffect(() => {\n  const fetchUsers = async () => {\n    try {\n      setLoading(true);\n      const response = await fetch('/api/users', {\n        headers: { Authorization: `Bearer ${token}` }\n      });\n      const data = await response.json();\n      setUsers(data);\n    } catch (err) {\n      setError(err.message);\n    } finally {\n      setLoading(false);\n    }\n  };\n  \n  fetchUsers();\n}, []);\n```\n\n### After (New Way)\n```jsx\n// ✅ Do this instead\nconst { data: users = [], isLoading, error } = useUsers();\n```\n\n### Migration Steps\n\n1. **Replace fetch/axios calls** with the appropriate hook\n2. **Remove useState for data, loading, error** - hooks provide these\n3. **Remove useEffect for data fetching** - hooks handle this automatically\n4. **Use mutation hooks** for POST/PUT/DELETE operations\n5. **Remove manual cache invalidation** - mutations handle this automatically\n\n## 🔍 Debugging\n\n### React Query Devtools\nThe devtools are automatically enabled in development mode. Press `F12` and look for the React Query tab.\n\n### Query Keys\nAll query keys are centralized in `api/config.js`:\n```jsx\nimport { QUERY_KEYS } from '../api/config';\n\n// Use these for manual cache operations\nqueryClient.invalidateQueries({ queryKey: QUERY_KEYS.USERS.ALL });\nqueryClient.setQueryData(QUERY_KEYS.ASSOCIATIONS.BY_ID('123'), newData);\n```\n\n## 🚨 Important Notes\n\n1. **Never call API functions directly in components** - always use hooks\n2. **Mutations automatically invalidate related queries** - no manual refresh needed\n3. **Queries are cached and shared** - multiple components using the same hook share data\n4. **Loading and error states are handled automatically**\n5. **Authentication is handled by axios interceptors** - no need to pass tokens manually\n\n## 📖 Example Component\n\nSee `components/AssociationsExample.tsx` for a complete example showing:\n- Data fetching with loading/error states\n- Create, update, delete operations\n- Form handling\n- Optimistic updates\n- Error handling\n\nThis architecture provides:\n- ✅ Automatic caching and background updates\n- ✅ Optimistic updates\n- ✅ Error handling and retry logic\n- ✅ Loading states\n- ✅ Automatic refetching on window focus/reconnect\n- ✅ Request deduplication\n- ✅ Centralized API configuration\n- ✅ Type safety (when using TypeScript)\n- ✅ Developer tools for debugging
